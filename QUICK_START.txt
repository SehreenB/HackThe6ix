================================================================================
APOGEE CANADA — QUICK START GUIDE
================================================================================

You have everything you need to build and deploy Canadian surgical access tool.

WHAT'S IN THIS ZIP:
✓ All backend Python code (FastAPI, optimizer, endpoints)
✓ Mock Canadian data (4,750 DAs, 177 EDs, realistic travel times)
✓ Complete documentation (README, guides, API reference)

✓ Frontend reference (from original Liberia version - for comparison)

================================================================================
STEP 1: EXTRACT & SETUP (5 minutes)
================================================================================

1. Extract this zip file to a folder

2. Install Python dependencies (one-time):
   
   On Mac/Linux:
   $ pip install fastapi uvicorn pandas numpy pydantic python-dotenv anthropic
   
   On Linux (Ubuntu):
   $ pip install fastapi uvicorn pandas numpy pydantic python-dotenv anthropic --break-system-packages

3. Create .env file in apogee_canada/backend/:
   
   ANTHROPIC_API_KEY=sk-your-key-here
   
   (Get your API key from https://console.anthropic.com/account/keys)

================================================================================
STEP 2: TEST LOCALLY (5 minutes)
================================================================================

1. Open terminal and navigate to the backend folder:
   $ cd apogee_canada/backend

2. Start the server:
   $ python -m uvicorn main:app --reload
   
   You should see:
   Uvicorn running on http://127.0.0.1:8000

3. In another terminal, test an endpoint:
   $ curl http://localhost:8000/api/data
   
   Should return JSON with Canadian DAs and EDs data

4. View interactive docs (fancy):
   Open http://localhost:8000/docs in your browser

================================================================================
STEP 3: DEPLOY (10 minutes)
================================================================================

A. RENDER.COM (Easiest):

   1. Go to https://render.com (free account)
   2. Click "New +"  →  "Web Service"
   3. Connect your GitHub repo (push backend/ folder)
   4. Set environment variable:
      - Key: ANTHROPIC_API_KEY
      - Value: sk-your-key
   5. Click "Deploy"
   6. Wait ~2 minutes
   7. Your deployed URL appears (e.g., https://apogee-xyz.onrender.com)

B. RAILWAY.APP (Alternative):

   1. Go to https://railway.app (free account)
   2. Click "New Project"
   3. Connect GitHub repo
   4. Set ANTHROPIC_API_KEY environment variable
   5. Deploy
   6. Your URL appears

================================================================================
STEP 4: TEST DEPLOYED ENDPOINTS
================================================================================

Replace YOUR_URL with your deployed URL:

$ curl https://YOUR_URL/api/data
$ curl https://YOUR_URL/api/workforce-alignment
$ curl https://YOUR_URL/api/seasonal-access
$ curl https://YOUR_URL/api/investment-analysis

$ curl -X POST https://YOUR_URL/api/optimize \
  -H "Content-Type: application/json" \
  -d '{"n": 5}'

If these work, you're done! Share the URL with Betul.

================================================================================
STEP 5: (OPTIONAL) UPGRADE TO REAL DATA
================================================================================

Once deployed with mock data working:

1. Download real McGaughey datasets from figshare (see download_datasets.py)
2. Run: python join_datasets.py
3. This creates: backend/data/da_working_dataset.csv (real Canadian data)
4. Commit & push to GitHub
5. Render auto-redeployes
6. Now using real data!
