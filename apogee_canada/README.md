# Apogee Canada: Surgical Access in Rural and Remote Communities

A data-driven tool for planning rural and remote surgical facility placement across Canada, powered by real population data, travel-time analysis, and machine learning.

**Team**: Betul Cetintas (frontend / Base44) + Sehreen Basara (backend / deployment / Freesolo)  
**Event**: Hack the 6ix 2026  
**Status**: Phase 0 (Data Pivot) ✓ Complete | Phase 1 (Foundation) → In Progress

## Quick Start (Local Testing)

### 1. Generate Mock Data
```bash
python generate_mock_data.py
# Creates backend/data/ with 4,750 Canadian DAs, 177 EDs, realistic travel times
```

### 2. Install Backend Dependencies

```bash

```

```bash

curl http://localhost:8000/api/data

curl -X POST http://localhost:8000/api/optimize \

# Get analysis endpoints
curl http://localhost:8000/api/investment-analysis
curl http://localhost:8000/api/workforce-alignment
curl http://localhost:8000/api/seasonal-access
```

View interactive docs at http://localhost:8000/docs

## Key Features

### 1. Optimizer
Greedy algorithm for discrete dissemination area (DA) facility placement:
- **Input**: Candidate DAs with population + travel time to nearest ED
- **Algorithm**: Iteratively place facilities at locations with highest unserved population gain
- **Output**: Recommended locations + updated coverage statistics + coverage curve

**Differences from Liberia version**:
- Liberia: Raster-based (pixels, skimage contours)
- Canada: Discrete DA-level (pandas, haversine distance)
- Same JSON response format for Base44 compatibility

### 2. Data Endpoints
- **`/api/data`** — Initial population + travel times + coverage stats
- **`/api/investment-analysis`** — Illustrative provincial facility investment cases (Ontario, Manitoba)
- **`/api/workforce-alignment`** — CIHI physician supply by province + rural gaps
- **`/api/seasonal-access`** — Winter road closures, ice road access, First Nations isolation

### 3. Policy Brief Generator
Claude-powered brief generator customized for Canadian provincial health authorities:
- Emphasizes: rural/remote access, Indigenous health equity, telemedicine
- Generates training data for Freesolo fine-tuning (Phase 2)
- Fallback hardcoded brief if API unavailable

### 4. Freesolo Training (Phase 3)
- Generate 75–150 Canadian policy briefs across varied scenarios
- Fine-tune on Canadian framing + terminology
- Deploy fine-tuned model to `/api/brief-finetuned` in Phase 4

## Data Sources (Real vs. Mock)

### Mock Data (Ready Now)
Generated by `generate_mock_data.py`:
- 4,750 Canadian DAs (population, coordinates)
- 177 Emergency Departments
- Realistic travel times (haversine + 1.7x road factor)
- ~49% of population beyond 2-hour threshold

**Good for**: Testing, development, demo rehearsal

### Real Data (For Deployment)
Download from:

1. **McGaughey & Peters 2024** (figshare)
   - DA population centroids
   - DA-to-ED travel times
   - ED locations
   - CSD linking tables
   - **License**: CC BY-NC-ND (use as computation input, don't redistribute)

2. **Statistics Canada** (2021 Census)
   - DA boundary shapefiles
   - CSD boundary shapefiles

3. **CIHI** (Canadian Institute for Health Information)
   - Physician supply by province (2024)

4. **Salles & Mullan** (UK Polar Data Centre)
   - Ice road and winter road shapefiles

**Process**:
1. Download files to `data/` directory
2. Run `join_datasets.py` to merge on DAUID
3. Swap mock CSVs for real ones in `backend/data/`
4. Optimizer automatically loads new data

## API Reference

### GET `/api/data`
Returns initial Canadian DA population and ED locations.

**Response:**
```json
{
  "stats": {
    "total_pop": 5538579,
    "within_2hr": 2812000,
    "beyond_2hr": 2726579,
    "pct_covered": 50.8,
    "pct_beyond": 49.2
  },
  "das": [...],
  "eds": [...],
  "metadata": {...}
}
```

### POST `/api/optimize`
Greedy facility placement optimizer.

**Request:**
```json
{"n": 5}
```

**Response:**
```json
{
  "locations": [
    {
      "lat": 48.123,
      "lng": -81.456,
      "pop_gained": 185000,
      "province": "Ontario"
    },
    ...
  ],
  "updated_stats": {
    "total_pop": 5538579,
    "within_2hr": 3250000,
    "pct_covered": 58.7,
    ...
  },
  "coverage_curve": [
    {"n_facilities": 1, "pct_covered": 53.2, "pop_gained": 250000},
    ...
  ],
  "da_coverage_map": [...]
}
```

### POST `/api/brief`
Stream a policy brief for the given scenario.

**Request:**
```json
{
  "stats": {...},
  "optimizer_results": [...]
}
```

**Response**: Plain text, streamed

### GET `/api/investment-analysis`
Provincial investment framework (illustrative).

### GET `/api/workforce-alignment`
CIHI physician supply and rural workforce data.

### GET `/api/seasonal-access`
Winter road access and ice road patterns.

## Architecture: Liberia → Canada

| Component | Liberia | Canada |
|-----------|---------|--------|
| **Population** | WorldPop 2020 raster | StatsCan 2021 DA centroids |
| **Travel time** | Malaria Atlas raster | McGaughey real road times |
| **Optimizer** | Pixel-based (40×30 grid) | DA-based (4,750 units) |
| **Hospitals** | 52 hardcoded from OSM | 177 from McGaughey dataset |
| **Contour** | Raster mask | DA-level choropleth |
| **Workforce** | Hardcoded registry | CIHI provincial data |
| **Seasonal** | Rainy season | Winter ice roads |
| **Brief** | Liberia Ministry of Health | Canadian provincial health authority |

**Key Change**: Liberia used raster math (skimage, rasterio). Canada uses discrete DA-level data (pandas, spatial joins). Same API interface for Base44.

## Known Limitations

### Data
- **Mock data**: Realistic but synthetic (for testing only)
- **Investment analysis**: Illustrative (2 provinces only; no centralized Canadian surgical burden database)
- **Territories**: Yukon/NWT/Nunavut excluded (known gap, acknowledged in submission)

### Seasonal Access
- **Ice roads**: Salles & Mullan data pending; currently illustrative
- **Climate change**: Risk assessments included; actual climate impact modeling future work

### Workforce
- **Granularity**: Provincial-level (CIHI data); no DA-level physician distribution available
- **Rural recruitment**: Framework defined but not computed; requires consultation

## Troubleshooting

### Mock data generation fails
```bash
# Install dependencies
pip install pandas numpy

# Regenerate
python generate_mock_data.py
```

### FastAPI server won't start
```bash
# Check dependencies
pip install fastapi uvicorn

# Try port 8001 if 8000 is in use
python -m uvicorn main:app --port 8001 --reload
```

### `/api/brief` returns fallback text
- `ANTHROPIC_API_KEY` not set in environment
- Claude API rate limited or down
- Check `.env` file in backend/

### `/api/optimize` returns empty locations
- All DAs already within 2-hr threshold (no unserved population)
- Try larger `n` value
- Check that mock data loaded correctly
